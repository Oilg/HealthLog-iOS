# HealthLog iOS — правила работы

## Агентная система

**Для всех задач разработки — немедленно запускать агента `orchestrator`**, не выполнять задачу самостоятельно. Оркестратор управляет командой из 11 субагентов (архитектор, Swift разработчик, QA, безопасность, дизайн и др.) и ведёт задачу от ветки до TestFlight.

Исключения — выполнять напрямую, без orchestrator:
- Вопросы о коде: объяснить, найти, прочитать
- Срочные однострочные хотфиксы с явным указанием пользователя

**ВАЖНО:** любая задача разработки — редактирование файлов, коммиты, PR — только через оркестратора. Самостоятельное выполнение запрещено, даже если кажется что задача простая.

## Ветки

- Перед началом любой задачи проверить наличие активной невмерженной ветки (`git branch -a`, `gh pr list`).
- Если активной ветки нет — создать новую от `master`.
- Одна задача — одна ветка.

## Ревью (обязательно перед открытием PR)

**Запустить субагента-ревьюера до открытия PR. PR нельзя открывать пока ревью не завершено.**

- Запустить `/code-review` на ветке.
- Если ревьюер нашёл критические замечания — исправить, затем повторить ревью.
- Только после получения одобрения от ревьюера — открывать PR.

## Деплой

- После открытия PR — дождаться прохождения CI, затем **смержить PR без ожидания команды пользователя**: `gh pr merge <N> --squash`.
- После мержа — следить за CI: `gh run list --repo Oilg/HealthLog-iOS --limit 3`. Убедиться что все проверки прошли (`conclusion: success`).

## Линтер и форматирование

- SwiftLint и SwiftFormat проверяются в CI — код должен проходить до мержа.
- Строка до 160 символов.
- Имена идентификаторов минимум 2 символа.
- Файл до 400 строк (warning).

## Тесты

- Покрытие unit-тестами ≥ 15% (CI gate).
- После добавления нового кода — добавить тесты.
